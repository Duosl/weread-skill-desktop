- 新增数据范围选择能力，可指定生成报告时使用的阅读数据时间范围。
- 新增原文权限三档策略，用户可灵活控制是否允许报告使用划线和想法原文。
- 新增报告生成配置面板、状态卡片、日志面板和确认弹窗组件。

- 重构高级报告模块，优化整体交互体验与权限策略。
- 优化模板卡片展示，提供更直观的选择体验。
- 重构基础报告模板，将原文摘录替换为笔记信号统计，更符合权限策略。
- 标准化报告 HTML 渲染，为不同模板提供专属样式。
- 优化文案表述，统一使用"划线与想法"描述用户笔记内容。
- 重构页面组件传参，简化 props 传递逻辑。
- 新增 PageShell 返回按钮功能，支持页面级工作台导航。

### 修复

- 完善智能体模板 prompt 与输出校验规则，提升生成质量。
- 移除冗余的导出相关 hook 与类型定义，简化代码结构。

## [v0.2.0] - 2026-05-21

### 新增

- 新增独立「阅读报告」页面，支持从微信读书数据生成本地 HTML 阅读报告。
- 新增 3 个基础报告模板：阅读分析报告、读书旅程、年度阅读报告。
- 新增智能体报告模板能力，支持阅读人格分析、知识结构盲区、下一阶段阅读建议等 AI 叙事型报告。
- 新增本地 Agent CLI 桥接能力，可检测并调用本机可用的 Claude Code、Codex、Gemini 等命令行 Agent。
- 新增智能体报告任务中心，支持后台生成、取消任务、查看生成状态、打开历史报告和删除历史记录。
- 新增智能体报告工作区协议：每个任务独立写入 input、data、output 和任务元数据，报告结果输出为 HTML。
- 新增笔记工作台，将笔记浏览与 Markdown 导出合并到同一入口，并保留旧导出路由跳转。

### 优化

- 阅读报告页改为「基础模板 / 智能体模板」双 Tab，默认进入智能体模板目录。
- 智能体模板交互简化为选择模板后直接生成，模型与 CLI 使用用户本机默认配置。
- 智能体生成过程改为模型输出流展示，并支持简洁 / 详细两种显示模式。
- 书架筛选简化为「全部 / 已读完」，并新增本地一级分类筛选。
- 书籍卡片展示完整分类；未分类书籍显示「未分类」。
- 笔记浏览页支持从当前书直接进入导出流程。
- HTML 报告预览与导出复用同一数据模型，预览写入应用私有目录，正式导出写入用户选择目录。

### 修复

- 修复智能体报告声明可使用个人划线和想法但未实际提供原始笔记数据的问题；授权后会写入 `data/notes.raw.json`。
- 修复 `agent-cli-bridge` 使用仓库外相邻路径导致干净 checkout 或 CI 构建失败的问题；现在已 vendoring 到仓库内。
- 强化报告与导出文件写入校验，避免空内容或写入失败被误判为成功。
- 优化微信读书 API 缓存控制，报告强制刷新时可同步刷新划线和个人想法数据。

### 文档

- 新增高级阅读报告实现方案文档。
- 更新 MVP 设计文档、需求池和 UI 规范，补充阅读报告、智能体模板、工作区协议和当前开发边界。
