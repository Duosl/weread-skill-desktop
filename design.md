# 书迹设计执行说明

最后更新：2026-05-28

本文件是 UI 任务的执行入口。长期视觉原则见 `ui-style-guide.md`，产品范围见 `mvp-design-doc.md`。做 UI 改动时必须同时查看当前页面代码和 `src/index.css`、`src/styles/pages/*.css`。

## 当前 UI 事实

当前应用已经形成以下主界面：

- 左侧固定导航 + 顶部工具栏 + 主工作台。
- 概览页使用阅读数据 hero、统计卡片、趋势图和榜单。
- 书架页支持列表和封面墙，详情以侧向面板展开。
- 划线与想法页是工作台，顶部在「浏览 / 导出」之间切换。
- 分享卡片使用右侧抽屉式工作台。
- 报告页包含基础模板、高级模板、任务状态、历史记录和报告预览。
- 连接器和设置页使用较克制的表单/面板结构。

## 当前问题

优先级从高到低：

1. `src/index.css` 仍承载大量全局、页面和组件样式，虽然已有 `src/styles/pages/report.css`、`settings.css`，但样式分层还不完整。
2. Button、IconButton、Card、SegmentedControl 已有基础组件，但页面里仍有部分专用 action class。
3. 报告页层级复杂，仍是最容易发生视觉漂移的区域。
4. 弹窗/抽屉层级需要继续统一 z-index、焦点管理和操作区密度。
5. 用户可见文案需要持续消除内部字段和技术概念。

## 设计 token 方向

继续使用有限 token，不在页面组件中散落任意颜色、阴影和半径。

建议分层：

```text
src/styles/
  tokens.css
  base.css
  layout.css
  components.css
  pages/
    overview.css
    shelf.css
    notes-workbench.css
    report.css
    settings.css
```

当前如果暂不拆文件，至少在 `src/index.css` 内保持同样分区。

## 组件收敛规则

- 新增按钮优先使用 `Button` 或 `IconButton`。
- 新增互斥选择优先使用 `SegmentedControl`。
- 新增普通容器优先判断是 Surface、Panel、Card、Paper 还是 Modal，不随手创建新卡片体系。
- 报告页可以有局部表现力，但动作按钮、弹窗、错误态和任务状态仍要回到全局组件语义。
- 专用 class 只负责布局和少量局部差异，不重写主色、危险色、基础阴影和基础圆角。

## 页面执行清单

### 概览

- 保持“阅读总时长”作为首屏主记忆点。
- 趋势图支持逐层查看年/月/日，面包屑和加载态要清楚。
- 分类偏好、最长阅读和基础数字不与核心趋势抢层级。

### 书架

- 列表视图适合扫描，封面墙适合浏览。
- 书籍详情从右侧展开，展示进度、笔记数量和跳转/导出动作。
- 不新增书籍详情页，不扩成阅读器。

### 划线与想法

- 内容正文比工具栏更重要。
- 筛选和搜索应服务于快速定位，不制造过多模式。
- 分享入口必须可见但低强调，不能只靠 hover 发现。

### 导出

- 保持工作台结构：范围 -> 内容选项 -> 预览 -> 生成/同步。
- 单本预览要像真实 Markdown 文件，多本导出不伪造合并预览。
- 成功结果要告诉用户文件或知识库位置。

### 报告

- 模板卡用于选择，不把所有导出和打开动作铺满列表。
- 高级模板必须说清是否会读取划线原文和个人想法。
- 任务历史是档案，不是聊天流。
- HTML 报告预览要接近真实文件或演示页，不做过度装饰。

### 设置与连接器

- 权限、缓存、统计、凭证等说明必须写成普通用户能理解的语言。
- 凭证字段默认脱敏，显示/隐藏要有明确按钮和标签。
- ima 同步说明要直接说“会把选中的 Markdown 笔记导入到你选择的知识库”。

## 落地页方向

本轮落地页采用“编辑部式阅读档案桌面”方向：

- 首屏左侧是品牌与价值主张，右侧是产品工作台的静态视觉重构。
- 背景使用纸张纹理、细分割线和书脊式色条，不使用泛 AI 渐变。
- CTA 保持下载和查看源码两项。
- 页面节奏：Hero -> 功能证据 -> 工作流程 -> 隐私边界 -> 下载。
- `landing/index.html` 保持单文件，允许引用仓库内 `public/shuji-icon.png`。

## 验收清单

- 375px、768px、1440px 宽度下无横向滚动。
- 首屏标题、CTA、产品视觉不会互相遮挡。
- 字体不随 viewport 宽度任意缩放。
- 主要按钮可键盘聚焦。
- 图标或图片有可读替代文本，装饰图可留空。
- 文案不出现内部字段名。
- `./init.sh` 通过。
