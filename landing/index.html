真实能力，不做空壳展示

从微信读书取回数据，再变成你能长期使用的资料。

书迹围绕“整理、导出、复盘、分享”设计，不进入在线阅读、书城推荐或公共社交。它只做一件事：把你已经读过、划过、想过的内容变得可追溯。

01

私人书架账本

同步书架、阅读进度、最近阅读和笔记数量。列表适合扫描，封面墙适合回看自己的阅读轨迹。

02

笔记工作台

按书籍查看划线和个人想法，支持搜索、类型筛选、颜色筛选，并从同一处完成 Markdown 导出。

03

报告与分享

生成基础阅读报告，也可以在明确授权后生成高级报告。单条划线和想法可保存为分享卡片。

桌面工作流

从授权到归档，路径尽量短。

所有关键动作都在本地桌面端完成。你选择读取什么、导出什么、同步到哪里，书迹负责把过程做清楚。

配置微信读书 API Key

设置页会说明读取范围和保存位置，凭证默认脱敏显示。

刷新书架、统计和笔记

概览、书架、划线与想法进入同一个阅读档案工作台。

隐私边界

先说清楚会读取什么，再让你决定是否继续。

书迹面向普通用户写权限说明，不要求你理解内部字段或命令。涉及划线原文、个人想法和高级报告时，会把数据边界摆在操作前。

微信读书 API Key、ima 凭证和导出目录保存在本机配置里，前端只展示脱敏信息。
Markdown、HTML 报告和分享图片由你主动导出到本地目录，不自动发布到线上。
高级报告涉及原文证据时需要明确确认，未授权时不会假装拥有划线原文或个人想法。
匿名统计只用于安装规模和版本分布，不采集微信读书内容、API Key、文件路径或用户操作事件。

把散落在微信读书里的痕迹，收回自己的资料库。

书迹当前处于 Post-MVP 收敛阶段：核心导出、报告、分享卡片和 ima 同步链路已经可用，后续改进将围绕真实用户反馈继续打磨。
